Dokumentation av API-Bas Server

Routes

G-delen

Läser från databasen:

  • GET /users - returnerar alla användare.
  • GET /users/{id} - returnerar en user med angivet id eller status 404 om användaren saknas.

Skriver i databasen:

  • POST /users - skapar en ny användare accepterar JSON objekt på formatet {"firstname": "", "lastname": "", "username": "", "password": "", "email": ""}. userId är obligatoriskt.

Uppdaterar data med PUT:
