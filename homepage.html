Dokumentation av det här APIet

Routes

Läser från databasen:

  • GET /users - returnerar alla användare.
  • GET /users/{id} - returnerar en user med angivet id eller status 404 om användaren saknas.

Skriver i databasen:

  • POST /users - skapar en ny användare accepterar JSON objekt på formatet {"firstname": "", "lastname": "", "userId": "unikt namn", "passwd": ""}. userId är obligatoriskt.
