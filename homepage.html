Dokumentation av API-Bas Server

Routes

    Skapa användare:

  • POST /users
    - skapar en ny användare med hashat lösenord, accepterar JSON objekt på formatet {"firstname": "", "lastname": "", "username": "", "password": "", "email": ""}. Alla fält är obligatoriska.

    • Logga in:

  • POST /login
    - inloggning, accepterar JSON-objekt på formatet {"username": "", "password": ""} och den skickar tillbaka en token.

    • Ser alla användare:

  • GET /users
    - returnerar alla användare. Endast för autentiserade användare.

    • Ser specifikt användare:

  • GET /users/{id}
    - returnerar en user med angivet id eller status 404 om användaren saknas. Endast för autentiserade användare.

    • Uppdatera användare:

  • PUT /users/{id}
    - uppdaterar användaren med angivet id, accepterar JSON-objekt på formatet {"lastname": "", "username": "", "password": "", "email": ""}. Endast för autentiserade användare.
